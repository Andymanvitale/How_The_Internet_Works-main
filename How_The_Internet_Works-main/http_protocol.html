HTTP/HTTPS

HTTP stands for Hyper Text Transfer Protocol and HTTPS just adds secure to end end of that. This is essentially the way that your computer knows what to do when it goes to a website. For example, if I was to go to a website my computer would send many "get requests" to it in order to do what I ask it to do. When I want to search something, I would send a post request and it would come back with whatever the answer was.

Since there is potentially personal data being sent across networks, someone could access it without permission. The way that was developed to counteract this is by creating a secure channel for communications. HTTPS uses trusted certificates to ensure that you are on the correct website and your information is not being taken from you. While also doing that, your computer uses SSL and TLS which is secure sockets layer and transport layer security. These all run together to ensure that you are staying safe while browsing, but it can only go so far in some instances. To ensure that you are being protected even further, encryption and decryption is constantly used. The standard for modern day encryption is 256 bit keys which are secure enough for most people to remain safe during daily use. One way this is done is by SSL and TLS using public key encryption. This method ensures that they key can be accessed and data is sent reliably and safely, but also with high speeds. Asymmetric keys are used so people can send messages and the recipient can receive a message without having to know the encryption key of the person, but they have their own decryption key which receives the messages meant for them.

Looking at this example shows how a public key can send a message across a network while remaining safe. Neither one of the people know the key of the other person but they were both able to communicate.
